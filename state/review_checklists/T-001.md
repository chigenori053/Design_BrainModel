# T-001 Self-Audit Checklist（現行版）

現在の開発・運用で継続的に必要な確認項目だけを残したチェックリスト。
一時対応向けの項目や、現時点で運用していない項目は削除済み。

## 必須（毎回実施）

- [ ] 変更対象に対するテスト（unit / integration）が追加または更新されている
- [ ] 変更対象クレートのテストがローカルで成功している
- [ ] 主要な決定性テスト（determinism系）に影響がある場合、再実行して成功している
- [ ] 既存の公開インターフェースを壊す変更がない（壊す場合は明示し、移行手順を記載）
- [ ] 新規の技術的負債が発生した場合、`state/TASK_STATE.yaml` か関連ドキュメントに記録している

## 条件付き（該当時のみ実施）

- [ ] CI heavy 相当の経路に変更がある場合のみ、heavy テストを実行している
- [ ] ベンチマーク/性能に影響する変更の場合のみ、回帰確認を行っている
